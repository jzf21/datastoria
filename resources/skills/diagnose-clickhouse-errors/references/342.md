# Code 342: METADATA_MISMATCH

The table metadata in the Keeper does not match local metadata.

## Workflow

1. Extract these three parts from the DB::Exception text when present:
   - `stored in ZooKeeper: ...`
   - `parsed from ZooKeeper: ...`
   - `local: ...`
2. Treat this as a cluster metadata consistency issue (usually not a user SQL syntax issue).
3. Present:
   - error `342` / `METADATA_MISMATCH`
   - the table name if present in the message
   - the extracted values for `stored in ZooKeeper`, `parsed from ZooKeeper`, and `local` (if any part is missing, explicitly say "not found in error text")
4. In Fix, instruct the user to escalate to the cluster maintainer/SRE and include the extracted metadata fragments so they can investigate Keeper and replica state.
5. Omit Example unless the user explicitly asks for a read-only diagnostic query.
