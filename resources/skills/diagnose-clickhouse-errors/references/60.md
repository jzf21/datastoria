# Code 60: UNKNOWN_TABLE

The query references a table that does not exist or cannot be resolved in the current scope.

## Workflow

1. Extract the table name from the error text.
   - If it appears as `database.table` (qualified), extract both parts.
   - If it appears as a bare name (unqualified, e.g. `'distribution_queue'`), note that no database was specified — ClickHouse will have searched `currentDatabase()`.
2. Common causes:
   - typo in the table name
   - wrong or missing database qualifier
   - table exists in another database
   - table was recently dropped or renamed
3. **If the table name is qualified**, call `execute_sql` with the SQL below, replacing `DATABASE_NAME` and `TABLE_NAME` with single-quoted strings. Do not modify the query or call `execute_sql` again.

    ```sql
    SELECT
        name,
        engine,
        levenshteinDistance(lower(name), lower('TABLE_NAME')) AS dist
    FROM system.tables
    WHERE database = 'DATABASE_NAME'
    ORDER BY dist
    LIMIT 3
    ```

4. **If the table name is unqualified**, call `execute_sql` twice in sequence, replacing `TABLE_NAME` with a single-quoted string each time.

   First call — get the current database:

    ```sql
    SELECT currentDatabase()
    ```

   Second call — find the closest table matches across all databases:

    ```sql
    SELECT
        database,
        name,
        engine,
        levenshteinDistance(lower(name), lower('TABLE_NAME')) AS dist
    FROM system.tables
    ORDER BY dist
    LIMIT 3
    ```

5. Interpret the result:
   - `dist = 0`: the table exists but in a different database than the one resolved at query time; suggest adding an explicit `database.table` qualifier.
   - `dist > 0`: likely a typo; list all returned matches with their `database` and `engine` so the user can pick the right one.
   - No rows: the table may not exist at all; suggest the user check `SHOW DATABASES` and `SHOW TABLES`.
6. If `execute_sql` fails, diagnose from the error text using the causes in step 2 as a checklist.
7. Present: error `60` / `UNKNOWN_TABLE`, the unknown table name, whether it is qualified or unqualified, whether it is likely a typo or a wrong-database issue, the most likely fix, and one fenced SQL example when the fix is clear.
